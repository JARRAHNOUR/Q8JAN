## Reporting a Security Vulnerability

**Security Contact**

<q8janproject@gmail.com>

If you discover a security issue related to the Q8JAN project, please report it privately.

Do not disclose security vulnerabilities publicly before they have been reviewed.

Reports should include:

- Description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if available)

---

## Scope

This policy applies to:

- Smart Contract
- Deployment Scripts
- Documentation
- Website

---

## Response Policy

Security reports will be reviewed as soon as possible.

Confirmed issues will be fixed before a new production release whenever applicable.

Security updates will be documented in the project changelog when appropriate, while sensitive technical details may be withheld until a fix is available.
