# Contributing to Q8JAN

Thank you for your interest in contributing to the Q8JAN project.

## Development Principles

- Keep the smart contract simple.
- Preserve the fixed supply model.
- Do not introduce hidden minting.
- Do not introduce transaction taxes.
- Maintain transparent documentation.

---

## Pull Requests

Before submitting changes:

- Verify compilation.
- Review documentation.
- Keep commits clean and descriptive.

---

## Code Style

- Clear naming
- Minimal complexity
- Security-first approach

---

## Reporting Issues

Please provide:

- Description
- Expected behavior
- Actual behavior
- Steps to reproduce

---

Thank you for helping improve Q8JAN.
