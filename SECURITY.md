# Security Policy

## Q8JAN Security Policy

The security of the Q8JAN project is a top priority.

We appreciate responsible security research and encourage anyone who discovers a potential vulnerability to report it privately before any public disclosure.

---

## Supported Versions

| Version | Supported |
|----------|-----------|
| 1.0.x | ✅ Yes |

---

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

Until the official security contact is published, security reports should remain private and should not be publicly disclosed.

Future reporting channels may include:

- Official Website
- Official Email
- GitHub Issues (if enabled for security reporting)

---

## Scope

This policy applies to:

- Smart Contracts
- Website
- Documentation
- Deployment Scripts
- Future Official Tools

Third-party software, wallets, exchanges, and blockchain infrastructure are outside the scope of this policy.

---

## Responsible Disclosure

Please do not:

- Publish vulnerabilities before they are reviewed.
- Exploit vulnerabilities for personal gain.
- Disrupt project services.
- Attempt unauthorized access to wallets or infrastructure.

Responsible disclosure helps protect both the project and its community.

---

## Security Principles

Q8JAN follows several security principles:

- Simplicity over unnecessary complexity.
- Fixed token supply.
- No hidden minting functionality.
- Transparent smart contract architecture.
- OpenZeppelin security standards.
- Continuous security improvements.

---

## Disclaimer

While every effort is made to maintain a secure ecosystem, no software can be guaranteed to be completely free from vulnerabilities.

Users are responsible for protecting their wallets, private keys, and devices.
